# 数据持久化字典：伊甸园的物理法则 (Sandbox Persistent Layer)

本系统在沙盒执行环境（Layer 2: 伊甸园 / Sandbox）内通过文件系统实现数据的持久化隔离。这组物理文件的相互咬合，构成了无状态生命体状态机迭代进化的闭环载体。

## 圣物体系（Artifacts Types）

### 1. `Bible.txt` (概念体/不可变指引)
- **用途**：上帝预设的世界架构观文书与高维目标指示。
- **访问控制**：**只读**。只能由高维度的 God/Host 向沙盒映射（Bind Mount 挂载为只读），Adam 绝对无权修改。
- **作用域**：全局且跨生命周期永久有效。

### 2. `Diary.txt` (时间记忆/事件追溯)
- **用途**：生命体思考与行动结果历史记忆的序列化堆栈。
- **访问控制**：**Append-Only 追加模式**。Adam 运行时只能产生并压入新段落。Host 原则上不对其做干预抹除。
- **作用域**：长期记忆沉淀，可能随沙盒持续运行导致尺寸无限大，需要 Adam 自主进化出索引浓缩能力。

### 3. `agent.py` (肉身形态/可变物理形态)
- **用途**：承载 Agent 启动加载和执行流的基础逻辑体。
- **访问控制**：**读写覆写**。Adam 单次觉醒结束前，可自主对其覆写更新进化形态（通过替换新生成的逻辑树）。这一动作会触发大天使的安眠与唤醒守护程序。
- **作用域**：每一次覆写保存的新版本，都作为触发重载生命下一纪元迭代触发点。

### 4. `error.log` (前世死因/梦魇警示)
- **用途**：当 Adam 本次执行流程报错导致容器生命周期终止时，遗留死因抛错信息的停放隔离区。
- **访问控制**：由神界的**大天使后台获取守护抛出栈并强制生成写死后，复活重铸沙盒并投入执行流**。供下一次重开的 Adam 规避相同死法。
- **作用域**：通常仅在崩溃前一次溯源时有意义，用于错误回退逻辑参考使用。
